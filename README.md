## Design System

A living design system documenting the principles, patterns, and components I use across projects.

- **Color tokens** — semantic naming, accessible contrast ratios (WCAG AA+)
- **Typography scale** — fluid type system with clear hierarchy
- **Spacing system** — 4pt/8pt grid for consistent layouts
- **Component specs** — interactive states, accessibility notes, usage guidelines

## Research Templates

Ready-to-use templates for common UX research activities:

- **User interview script** — semi-structured with probing questions
- **Usability test plan** — tasks, metrics, note-taking guide
- **Affinity mapping guide** — from raw notes to insights
- **Survey design checklist** — avoiding bias and leading questions
- **Research debrief template** — synthesizing findings for stakeholders

## My Design Process

```
01 Discover → 02 Define → 03 Design → 04 Test → 05 Ship
```

| Phase | Activities |
|---|---|
| **Discover** | Stakeholder interviews, competitive analysis, analytics review |
| **Define** | HMW statements, personas, journey maps, problem framing |
| **Design** | Wireframes, prototypes, design system components |
| **Test** | Usability testing, A/B experiments, accessibility audits |
| **Ship** | Dev handoff, QA review, post-launch measurement |

## Principles I Design By

1. **Clarity over cleverness** — if it needs explaining, it needs redesigning
2. **Accessibility is not optional** — design for everyone from the start
3. **Research before assumptions** — talk to real users, often
4. **Design systems save time** — consistency compounds
5. **Shipping beats perfecting** — iterate in the real world
